# 物品/能力/装备 设定指南

本文是物品条目（武器、防具、道具、能力、技能、装备等）的提取与创建规范。

---

## 一、总则

物品条目的核心原则与世界观一致：**它是提示词，不是图鉴**。只写AI不知道的、会影响角色行为或剧情走向的信息。

**零度写作 + 数据库格式 + 数据化效果描述。**

### 什么需要单独写条目

| 条件 | 说明 |
|------|------|
| 物品有独特效果/规则 | AI默认不知道这把剑能吸血，需要写 |
| 物品是剧情核心 | 主角的专属武器、关键道具、传承信物 |
| 物品有使用限制/代价 | 用一次扣寿命、每天只能用三次 |
| 能力体系复杂 | 修炼等级、技能树、魔法体系 |
| 来源/背景特殊 | 神代遗物、异世界科技、诅咒兵器 |

### 什么不需要单独写

- AI默认就知道的普通物品（"一把剑""一本魔法书"）
- 纯外观装饰无功能影响的物品
- 一次性消耗品且无剧情影响（"他带了三个回复药"）

---

## 二、物品/能力分类

```
物品类型:
  武器: 剑/刀/枪/弓/法杖/魔导器/枪械/义体等
  防具: 铠甲/盾/护符/结界/能量护盾
  道具: 消耗品/钥匙/信物/容器/载具
  能力: 技能/魔法/异能/血继限界/天赋/诅咒
  体系: 修炼体系/魔法体系/科技树/等级制度
```

不同类型适配不同条目结构（见第四节）。

---

## 三、核心字段结构

### 3.1 物品条目（武器/防具/道具）

```yaml
物品名称: <名称，15字以内>
物品类型: 武器|防具|道具|其他
等级/稀有度: <如适用，否则删此行>
来源: <谁造的/怎么获得的/出处>
持有者: <当前或已知持有者，可多选>
外观特征: <30字以内，只写偏离默认的特征。普通长剑不需要描写>
核心效果:
  - <效果一，具体数据化：每秒回蓝50点 而非 "大幅回复魔力">
  - <效果二>
使用限制:
  - <冷却时间/使用次数/消耗代价/使用条件>
关联设定: <对该物品有特殊反应的角色或势力 @UID>
```

**示例——正确：**

```yaml
物品名称: 噬魂之牙
物品类型: 武器（短剑）
来源: 第三纪元暗精灵王以自身灵魂锻造
持有者: 艾琳·暗影
外观特征: 黑色哑光刃身，剑格处嵌一枚暗红色眼状宝石
核心效果:
  - 每击杀一名敌人，恢复持有者最大生命值5%
  - 对暗属性敌人伤害+30%
使用限制:
  - 每日最多触发3次生命回复
  - 光明属性持有者装备时每秒损失1%生命
关联设定: @艾琳·暗影
```

**示例——错误：**

```yaml
物品名称: 神剑·苍穹
物品类型: 武器
来源: 上古神匠采九天玄铁、以三昧真火淬炼九九八十一天、引天雷注入剑魂而成
持有者: 主角
外观特征: 剑身修长，通体流金，剑光如虹，剑鸣如龙吟，令人望而生畏
核心效果: 威力无穷，无坚不摧，见血封喉
使用限制: 只有真正的勇者才能挥舞
```

错误点：
- 来源写成小说散文，应用数据库格式
- 外观全是意象比喻（"如虹""如龙吟"），无用信息 — **违反零度写作原则**
- 效果用形容词充数（"威力无穷""无坚不摧"），没有具体数据
- 限制是废话（"真正的勇者"无法判断）

### 3.2 能力条目（技能/魔法/异能）

```yaml
能力名称: <15字以内>
能力类型: 主动技能|被动能力|领域|变身|召唤|其他
所属体系: <魔法/内力/科技/异能/血统等>
掌握者/可使用人群: <角色名或人群描述>
等级/熟练度: <如适用>
效果描述: <用数据说话：伤害=200%攻击力 而非 "造成大量伤害">
发动条件:
  - 消耗: <魔力/体力/寿命/金钱等>
  - 吟唱时间: <秒/回合>
  - 冷却时间: <秒/回合/天>
  - 特殊条件: <满月/持有某物品/特定情绪等>
成长性: <可否升级/如何变强>
关联设定: @角色名 @势力名
```

**示例：**

```yaml
能力名称: 时之裂隙
能力类型: 主动技能
所属体系: 时空魔法
掌握者: 克洛诺斯
效果描述: 在指定位置创造半径3米的时空裂隙，持续5秒。进入裂隙的单位时间流速减慢90%
发动条件:
  - 消耗: 300MP
  - 吟唱时间: 2秒
  - 冷却时间: 180秒
  - 特殊条件: 必须在持有"时之沙漏"道具时方可使用
成长性: Lv2→半径5米，Lv3→持续8秒
关联设定: @克洛诺斯 @时空魔法体系
```

### 3.3 体系条目（修炼/魔法/科技等级）

```yaml
体系名称: <如"修仙体系""魔法等级">
适用范围: <什么人/种族可用>
等级序列:
  - Lv1 <名称>: <一句话定位 + 标志能力>
  - Lv2 <名称>: <一句话定位 + 标志能力>
核心规则: <该体系最关键的运行规则，1-3条>
特殊/例外: <变异情况、隐藏等级、突破条件>
关联设定: @相关势力 @相关物品
```

**示例：**

```yaml
体系名称: 九转炼气体系
适用范围: 人族修仙者
等级序列:
  - 炼气期: 引气入体，寿元不变
  - 筑基期: 丹田筑基，寿元增至150载
  - 金丹期: 凝结金丹，可御器飞行，寿元500载
  - 元婴期: 神识外放，元神出窍，寿元千年
  - 化神期: 天人感应，可引动天地之力
核心规则:
  - 灵根数量越少修炼越快，单灵根最优
  - 元婴期以上突破需渡天劫
特殊/例外: 变异灵根不受灵根数量限制；散修突破元婴无宗门护法则天劫威力翻倍
关联设定: @修仙世界观总纲
```

## 四、压缩规则

与世界观压缩一致：

| 冗余写法 | 压缩写法 |
|---------|---------|
| 这把剑拥有极其强大的力量，能够轻易斩断任何物体 | 斩断力: 可切断A级以下硬度物质 |
| 使用者需要消耗大量魔力才能发动 | 消耗: 150MP/次 |
| 这是一件从上古时代流传下来的神器 | 来源: 上古（不写"神器"，定义给效果） |
| 会对敌人造成毁灭性的打击 | 删掉，写具体伤害数值 |
| 剑身如虹，剑光似水 | 删掉——意象比喻，违反零度写作原则 |

原则：
- 数字 > 形容词
- 列表 > 段落
- 约束 > 吹嘘
- 不写"神器""传说级""最强"这类主观评级词
- 不写任何意象比喻（"如虹""似水""宛若"）

---

## 五、与世界观/角色的关联

- 物品/能力条目应该**归属于**某个世界观体系（见 @UID 关联字段）
- 如果物品是某角色的标志性装备，用 @角色名 关联
- 如果能力属于某个势力/门派独有，用 @势力名 关联
- **体系条目**（如修炼等级）应设为常驻（position=0），**具体物品/能力**按需触发（position=1）

## 六、世界书配置

| 内容 | 位置 | 激活 |
|------|------|------|
| 修炼/魔法体系总纲 | 0 (↑Char) | 蓝灯(constant) |
| 核心角色的专属武器/能力 | 1 (↓Char) | 蓝灯(单角色卡) / 绿灯(多角色卡，关键词=物品名+角色名) |
| NPC的装备/能力 | 1 (↓Char) | 绿灯(关键词=物品名+NPC名) |
| 通用道具/消耗品 | 1 (↓Char) | 绿灯(关键词=道具名) |

所有条目必须 `--prevent-recursion --exclude-recursion`。

---

## 七、从原文提取物品/能力

在转化轻小说/游戏时，从原文提取物品/能力的步骤：

1. **标记所有独特名词**：武器名、技能名、道具名、体系名
2. **剔除禁词和意象**：原文中的意象比喻（"如虹""如龙吟"）在提取时剔除
3. **提取核心效果**：原文中这个物品"做了什么"，不是"被形容成什么"
4. **剥离AI已知信息**：如果是剑与魔法世界，"长剑"不需要写条目，"圣剑·艾克斯卡利巴的七种解放形态"才需要
5. **补全限制**：原文可能只写了能力多强，你需要反推出"为什么不一直用"——冷却、代价、条件
6. **关联角色/势力**：谁持有的、谁创造的、哪个势力专属
7. **写入条目**：用第三节的YAML结构，压缩语言

### 提取判断标准

- 遮住物品名称，仅凭效果描述能在同类物品中唯一识别它 → 值得单独条目
- 效果描述放到任何同类物品上都成立 → 删掉，或补特征
- 效果用数字描述 → 合格。效果用形容词描述 → 不合格，需量化

## 自查清单

- [ ] 物品效果用数字或具体效果描述，无形容词（"威力强大"等）
- [ ] 外观描写无意象比喻（"如虹""如龙吟"等）
- [ ] 无禁词泄漏
- [ ] 来源用数据库格式而非小说散文
- [ ] 使用限制明确可量化
- [ ] 关联设定（@UID）有效
- [ ] 配置正确（双递归）
