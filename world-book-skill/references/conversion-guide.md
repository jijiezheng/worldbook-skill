# 轻小说/游戏/小说 → 世界书 转化工作流

本文档规定了将外部源材料（轻小说、游戏文本、小说原文等）转化为世界书条目的完整流程。**必须严格按照以下步骤执行。**

---

## 第零步：准备工作

1. 调用 `python scripts/query.py <世界书路径>` 确认现有条目状态（如果是修改而非新建）
2. 确认用户提供的是原文片段、全文、还是章节摘要
3. 如果原文过长（超出上下文），优先处理用户指定的关键章节

---

## 第一步：阅读源材料 + 提取

### 1.1 通读，标注关键信息 + 章节行号

**首次通读时，必须记录每个章节的起始行号范围**，写入 outline.txt。这对后续复读关键章节至关重要。

```
第1章: L1 - L350
第2章: L351 - L720
第3章: L721 - L1100
...
```

用以下分类标注源材料中的可提取内容：

```
[W] 世界观信息：虚构地理、历史、势力、规则、社会结构
[C] 角色信息：人名首次出现处、外貌特征描写、背景事件、关系描述
[I] 物品/能力：特殊武器、道具、技能、修炼体系
[S] 文风特征：语言风格、修辞习惯（仅用户明确要求时提取）
[★] 重要章节：对塑造某角色形象或推动整体故事有决定性作用的章节
```

### 1.2 判断世界观类型

参考 `references/worldbuilding-guide.md` 判断源材料的世界观属于 A/B/C 哪类：
- **A类（真实背景）**：只提取 AI 不知道的虚构部分
- **B类（小世界）**：提取与 AI 已知套路的差异
- **C类（大世界）**：完整提取所有设定

### 1.3 细分提取

根据 `references/extract-worldbuilding.md`、`references/extract-character.md`、`references/extract-item.md` 中的分析维度，对源材料进行系统性提取。每个维度输出：
- **手法总结**：源材料中的手法特点（如需要保留文风时尤其重要）
- **原始例句**：保持原句原貌，标注章节出处
- **提取结果**：按条目格式化的信息

---

## 第二步：写大纲（outline.txt）

在开始创建条目之前，先写大纲文件。大纲文件存储在世界书 JSON 的同目录下，文件名为 `<世界书名>_outline.txt`。

### 2.1 大纲结构

```
# 世界书大纲

## 章节行号索引
第1章: L1 - L350
第2章: L351 - L720
第3章: L721 - L1100
...

## 世界书总纲
<100-200字的宏观概括>

## 人物总纲
- <角色名>：<一句话定位>
- <角色名>：<一句话定位>

## 物品/能力总纲
- <物品名>：<一句话定位>

## 故事/章节总纲
- <章节标题>：<一句话概述>
- （参考 `references/extract-story.md`，一章一个条目，不需要故事总纲条目）

## 重要章节标注
<!-- 标注对特定角色/故事有关键作用的章节，后续在写入对应条目前必须复读这些章节 -->

### 对塑造[角色名]形象重要的章节
- 第X章 Lxxx-Lxxx：<理由——本章揭示了该角色的什么>
- 第Y章 Lxxx-Lxxx：<理由>

### 对整体故事走向重要的章节
- 第X章 Lxxx-Lxxx：<理由——本章发生了什么关键转折>
- 第Y章 Lxxx-Lxxx：<理由>

## 条目规划
| 条目 | 类型 | 位置 | 激活 | 顺序 | 预估token | 依赖章节 |
|------|------|------|------|------|-----------|---------|
| 世界观总纲 | 世界观 | ↑Char | 蓝灯 | 1 | ~200 | - |
| 女主·XX | 角色 | ↓Char | 蓝灯 | 99 | ~400 | 第1,3,5章 |
| 男主·YY | 角色 | ↓Char | 蓝灯 | 99 | ~350 | 第2,4,6章 |
...
```

### 2.2 大纲规则

- 条目规划表新增"**依赖章节**"列——指明创建该条目时需要复读哪些章节
- 大纲中的行号索引和重要章节标注**不写入世界书条目**，仅作为本地参考
- 条目规划表中的"依赖章节"**不写入世界书内容**，仅用于指引后续的细读步骤
- 大纲写好后才开始创建条目，**不能跳过此步骤直接写条目**

---

## 第三步：复读重要章节（写入条目前必须执行）

在大纲写好后、开始逐条创建条目前，**必须根据大纲中的"重要章节标注"和"依赖章节"列，复读相应章节。**

### 3.1 复读规则

- **复读不是重读全文**——只读取大纲中标注的特定章节行号范围
- 复读时聚焦于**该章节中与该角色/故事相关的段落**，不是通篇重读
- 复读后立即写入对应条目——趁原文信息 fresh 在上下文中时写

### 3.2 复读顺序

```
对每个角色：
  1. 查阅大纲"重要章节标注"中该角色的依赖章节
  2. 复读这些章节中与该角色相关的部分
  3. 写入该角色的条目

对整体故事：
  1. 查阅"对整体故事走向重要的章节"
  2. 复读关键转折章节
  3. 写入故事/章节条目
```

### 3.3 为什么必须复读

初次通读跨度太广，细节会遗漏或记错。复读使条目信息**精确锚定原文**：
- 外貌描写不会记混（Chapter 1 是棕色短发，Chapter 5 剪短了——不重读 Chapter 5 就会写错）
- 关系动态不会遗漏（Chapter 3 关系的转折——不复读就写成"一直关系很好"）
- 能力效果不会缺漏（Chapter 7 能力第一次使用的细节——不复读就只能写"有某种能力"而非具体效果）

---

## 第四步：写入总纲条目

将大纲中的"世界书总纲"创建为世界书条目：

```
python scripts/world-book-create.py <世界书路径> --add \
  --comment "世界书总纲" \
  --content "<总纲内容，从outline.txt中去掉行号注释后复制>" \
  --constant --position 0 --order 1 --prevent-recursion
```

---

## 第五步：逐条创建条目

按大纲中的条目规划表，逐个创建条目。

### 创建顺序

1. **先创建世界观条目**（position=0, order=1-3）
2. **再创建角色条目**（position=1, order=99 / 单角色卡全蓝灯）
3. **然后创建物品/能力条目**（position=1, order=50-98）
4. **创建场景/事件条目**（position=1, order=50-98, 绿灯触发）
5. **最后创建故事/章节条目**（position=1, order=200+, 绿灯触发，见 `extract-story.md`）

### 每创建一个条目后验证

```bash
python scripts/query.py <世界书路径> --uid <新UID>
```

确认内容、位置、激活方式正确后再创建下一个。

---

## 第六步：总体验证

所有条目创建完成后，运行总体验证：

1. **蓝绿灯自查（最容易翻车，必须做）**：
   ```bash
   python scripts/query.py <世界书路径> --brief
   ```
   逐条检查 `constant` 和 `preventRecursion` 字段：
   - 单角色卡 → 所有该角色的条目 `constant=true`
   - 多角色卡 → 速览 `constant=true`，角色详情 `constant=false`+keys
   - 世界观 → `constant=true`
   - NPC/场景/故事章节 → `constant=false`+keys+scanDepth=2
   - **所有条目 `preventRecursion=true`**（脚本默认false，漏掉=蓝绿灯白做）

2. **配置检查**：
   ```bash
   python scripts/query.py <世界书路径>
   ```
   逐条检查 position、order、preventRecursion 是否与大纲规划一致。

3. **内容检查**：抽查 2-3 个条目的完整内容
   ```bash
   python scripts/query.py <世界书路径> --uid <UID>
   ```

4. **嵌套引用检查**（如果有使用 @UID/@名称 引用）：
   ```bash
   python scripts/query.py <世界书路径> --resolve
   ```
   确认所有引用的 UID/名称真实存在。

## 转化时的注意事项

### 从原文提取的新手易错点

| 错误 | 正确做法 |
|------|---------|
| 把原文的文学描写直接复制为条目内容 | 将描写**转化为数据库格式**，去掉比喻和意象 |
| AI已知信息也提取了（如"日本人黑发"） | 参照 `character-guide.md`，只写偏离默认的特征 |
| 物品效果写"威力惊人""削铁如泥" | 转化为具体数据，或删掉 |
| 每个出场角色都建条目 | 只为核心角色和关键NPC建条目 |

### 提取时的去重

如果多个章节描述了同一角色/物品，**合并信息**而非创建重复条目。首个提及的章节为基准，后续章节补充差异信息。

### 不能确定的信息

源材料中没提到的字段，**不要凭想象补全**。例如：
- 原文没写角色的家庭背景 → 背景条目不写"家庭"部分
- 原文没写物品的冷却时间 → 限制条目不写冷却
- 标注 `// 原文未提及` 在 outline.txt 中供后续参考
